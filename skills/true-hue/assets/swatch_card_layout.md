# Swatch Card Layout Specifications

Use these layouts when the user opts in to a PDF swatch card deliverable. The PDF skill handles actual PDF generation — this file specifies what to hand to it.

Always ask the user before generating: *"Would you like me to generate a printable PDF swatch card you can save to your phone or print for shopping? I can make a full-page version, a wallet-sized card, or a phone wallpaper."*

---

## Layout A: Full-page swatch card (8.5" x 11" portrait)

### Purpose
Primary deliverable. Shopping reference users can keep on phone or print.

### Structure

**Top section (top 20% of page):**
- Header: "Personal Color Palette"
- Subheader: "[User's Name or 'Your'] Season: [Season Name]"
- Date of analysis
- Confidence level (one line)

**Middle section (middle 60% of page):**

Palette grid organized by role. Each color shown as:
- Swatch (approximately 1.5" x 1" rectangle)
- Color name below swatch
- Hex code below name

Grid layout, 5 columns wide:

- Row 1-2: **Neutrals** (8 colors)
- Row 3: **Lights** (6 colors) — may span 5 + 1 across rows
- Row 4-5: **Mids** (10 colors)
- Row 6: **Darks** (6 colors) — 5 + 1
- Row 7: **Accents** (4 colors) + **Statement** (2 colors) = 6 total

**Bottom section (bottom 20% of page):**

Two columns:

Left column — "Quick reference":
- Metals: [primary]
- "Black" substitute: [hex]
- "White" substitute: [hex]
- Face-framing top choices: [3 hex codes in small swatches]

Right column — "Avoid":
- 6-8 small swatches of avoid colors with mini-labels

Footer: small text "Generated by True Hue — verify colors in natural daylight before purchasing. Screens vary."

### Typography
- Heading font: bold sans-serif, 18-20pt
- Color names: regular sans-serif, 9pt
- Hex codes: monospace, 8pt
- Footer: 7pt italic

## Layout B: Wallet card (3.5" x 2" landscape)

### Purpose
Pocketable quick-reference while shopping. Business card sized.

### Structure

**Top strip (top 15%):** Season name + confidence indicator

**Main body (bottom 85%):**

Grid of 12 most-used colors as small swatches (roughly 0.4" x 0.4" each):
- 3 neutrals
- 3 mids
- 2 face-framing top choices
- 2 darks
- 1 statement
- 1 accent

Each swatch labeled with hex code only (no color names — too small).

Back side (if producing two-sided):
- "Your Avoid Colors" with 6 small swatches

### Typography
- Heading: 8pt bold
- Hex codes: monospace, 5-6pt

---

## Layout C: Phone wallpaper (9:16 aspect, 1080 x 1920 px)

### Purpose
Set as phone lock screen or wallpaper for constant at-a-glance reference.

### Structure

**Top 10%:** Season name in elegant typography, small

**Middle 75%:** Palette grid

- 4 columns x 8 rows = 32 swatches
- Organized vertically by role (neutrals at top, statement at bottom)
- Each swatch is roughly 270 x 180 px
- Hex code overlaid in small text (corner of swatch)
- Color name centered if space allows

**Bottom 15%:** 
- Small row of "avoid" colors (6 swatches)
- Tiny footer text with generation date

### Design considerations
- Should be visually attractive enough that the user actually uses it as a wallpaper
- Consider a subtle gradient or textured background between swatches (keep neutral so as not to contaminate color reading)
- Metal indicator (small icon or text) showing gold/silver/rose-gold preference

## Color rendering notes

Important constraints for the PDF generator:

1. **Use CMYK-friendly conversions** if user wants to print. Some saturated colors (especially Bright Spring/Winter brights) won't print true to screen on home printers. Include a note: "For exact color reference, use digital version; printed colors will approximate."

2. **Keep swatches large enough to be readable** — don't squeeze past a minimum of ~0.3" for any format. Below that, swatches become indistinguishable.

3. **Contrast text over swatches appropriately** — on very light swatches, text should be dark; on very dark swatches, text should be light. This is about *legibility of the label*, not color analysis.

4. **White space matters** — don't cram. The full-page layout should have clear margins and air between swatches. A crowded palette card is harder to use while shopping.

## Invocation pattern

### Prerequisite
Before first invocation in a session, read the PDF skill's current SKILL.md:

```
view /mnt/skills/public/pdf/SKILL.md
```

Path may differ in non-Anthropic environments. Adjust accordingly.

### Standard invocation (pseudo-structure)
The PDF skill accepts layout specifications and content. Use a prompt structured like:

> "Generate a PDF with these specifications:
> - Layout: [Full-page / Wallet card / Phone wallpaper] as described in assets/swatch_card_layout.md
> - Palette: [user's season]
> - Hex codes and roles: [list from the delivered palette]
> - Header: [user name or 'Your' + season name + date]
> - Output path: /mnt/user-data/outputs/[season-name]-swatch-card.pdf"

Read the actual PDF skill invocation requirements from its SKILL.md — the above is scaffolding, not literal syntax.

### Failure handling
If the PDF skill returns an error or is unavailable:
1. State the failure to the user: "PDF generation isn't available in this environment."
2. Produce a markdown-formatted palette summary with hex codes clearly listed in role groups
3. Suggest external conversion tools (do not recommend specific products; suggest category only: "any design tool that accepts hex codes, or a simple text-to-PDF converter")

### Multiple format requests
If the user requests multiple formats (e.g., "wallet card + phone wallpaper"), invoke the skill once per format. Each invocation uses its own layout specification. Do not attempt to batch formats into a single invocation — each has different dimensions and role organization.

## User options to offer

When asking about PDF generation, present the three layouts as choices:

> "I can generate three types of swatch card:
> 
> 1. **Full-page reference (PDF)** — printable, comprehensive, 30-40 colors with roles and hex codes. Best for home reference.
> 2. **Wallet card (PDF)** — pocketable 3.5x2 inch card with your top 12 colors. Best for shopping trips.
> 3. **Phone wallpaper (PNG)** — set as your lock screen to glance at while shopping online. Best for always-on reference.
> 
> Which would you like? I can generate any combination."

If the user says "all three," produce all three. If they say just one, produce just that one. If they say no, skip.
